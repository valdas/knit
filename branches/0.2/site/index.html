Welcome! Why are you Here?

At first glance, this site seems to be just some tools & tricks for AWK programmers.

But if that is all you see here, then you have not mastered the Art of AWK:

    To reveal simplicity, by stripping away the superfluous.

Programming in AWK is a study of the essential essence of a problem (and not trivial and tedious details of over-elaborated implementation tools). When you code in AWK, you are thinking about the problem, not the complexities of the programming languages (because there aren't any: no continuations, no inheritance, no pointers, no unions, etc).

AWK programs are very short to write and very quick to change. This brevity lets you rapdily build and evolve prototypes- which gives you the experience needed to uncover the correct design.

So AWK programming is a test of your understanding of a problem. When you really understand it, then everything suddenly becomes clear and clean and simple. Hence:

    The rule of AWK: keep AWKing till the code gets simple.

For example, there is no standard AWK development environment that supports development, testing, and documentation by a distributed community. So, we wrote one in AWK (of course!), with a little help from some Unix scripting tools:

  • Domain-specific languages for better design and and analysis;
  • Writing and documentation tools using literate programming;
  • Simplier builds with library management tools;
  • Continuous unit testing via test-driven development;
  • Data mining tools for better system testing;
  • Web-based content management (with version control) for all the above.
  • And we used the above to code up some sample applications.

What we found was that this was remarkably easy to do: this is a part time project and the first release of this site was built using 80 minutes per day for three months (includes writing, testing, documentation) - about 2 weeks full time work. Also, the resulting environment was not AWK-specific. For example, this site offers mappings of these tools into LISP, SMALLTALK, PROLOG, PHP, PYTHON,..., and (of course) AWK. So what started as a project to support AWK programming quickly became a project that supporting programming, in any language.

When you review this code, you might be surprised at how easy it is to code these tools. And that is the Art of AWK:

  • understanding something well enough...
  • ... so you can write it, just short enough, to get the job done.

And always remember: perfection is when there is nothing left to throw away1.
