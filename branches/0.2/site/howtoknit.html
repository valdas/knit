Variables

Naming conventions

Label all your global variables as MixedCase.

Label all your locals with a leading lowerCase letter.

This convention makes it easier to find escaped globals.

Globals

Are evil. Sometimes, a necessary evil. But nevertheless, evil. Remember, N-1 globals is always better than N.

Functions, not Patterns

Assume that most of your code will be AWK functions, not top-level functions.

This will make reusing it in other applications.

Applications

Applications are built in their own directory and have have

  • app.wak : the main driver
  • Makefile : the controller for building, testing, etc
  • etc/tests : a sub-directory for examples.
